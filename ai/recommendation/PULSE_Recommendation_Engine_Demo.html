Recommendation Engine

Core idea, equations, code, and a working interactive demo

📌 Core idea

The engine takes a lead's score (from the Lead Scoring module), how many days since its last activity, whether a reply has been received, and its current pipeline stage. It filters down to only the actions that make sense at that stage, scores each one with a transparent formula, and recommends the highest-scoring action — always paired with a plain-English reason.

Phase 1 is deliberately rule-based, not ML — this matches PULSE's "usable before it's automated" principle. The interface is designed so Stage 3's ML model can later replace the rule table without changing anything downstream.

📌 Feature inputs

FeatureTypeRole in the formula
current_scoreDerivedNormalized to 0–1, drives score-weighted actions
days_since_last_activityComputedConverted to an urgency factor (capped at 1)
reply_received_flagComputedBinary factor, shifts weight toward/away from follow-up
current_stageUser-enteredFilters which actions are even valid candidates

📌 Core equation

For every candidate action a valid at the lead's current stage:

weight(a) = w1 · scoreNorm + w2 · urgency + w3 · replyFactor

Where:

scoreNorm = score / 100 urgency = min(days_since_activity / 7, 1) replyFactor = reply_received ? 1 : 0 (or inverted, depending on the action)

The recommended action is the one with the highest weight:

recommended_action = argmax( weight(a) ) for a in validActionsForStage

The reason is generated from whichever single term contributed most to the winning action's weight — so the explanation is never invented, it's read directly off the math.

📌 Core logic (JavaScript)

// Each action defines which stages it's valid for, and how much
// weight it gives to score / urgency / reply. invertX flips the factor
// (e.g. "Mark as stale" favors LOW score, not high).
const actionDefs = {
  "Send follow-up":      { stages: ["Contacted","Qualified"],
                                w: {u:0.5, r:0.3, s:0.2}, invertR: true },
  "Schedule demo":       { stages: ["Qualified","Demo Scheduled"],
                                w: {s:0.6, u:0.2, r:0.2}, invertU: true },
  "Send proposal":       { stages: ["Demo Scheduled","Negotiation"],
                                w: {s:0.5, r:0.3, u:0.2}, invertU: true },
  "Mark as stale":       { stages: ["Contacted","Qualified","Demo Scheduled"],
                                w: {u:0.7, s:0.3}, invertS: true },
  "Escalate to manager": { stages: ["Negotiation"],
                                w: {s:0.5, u:0.5} }
};

function recommend(score, daysSinceActivity, replyReceived, stage) {
  const scoreNorm = score / 100;
  const urgency   = Math.min(daysSinceActivity / 7, 1);

  let results = [];
  for (const [name, def] of Object.entries(actionDefs)) {
    if (!def.stages.includes(stage)) continue; // not valid at this stage

    const sVal = def.invertS ? (1 - scoreNorm) : scoreNorm;
    const uVal = def.invertU ? (1 - urgency)   : urgency;
    const rVal = def.invertR ? (1 - (replyReceived?1:0)) : (replyReceived?1:0);

    let weight = 0, topFactor = "", topContribution = -1;
    if (def.w.s !== undefined) { const c = def.w.s*sVal; weight += c;
      if (c > topContribution) { topContribution = c; topFactor = "score"; } }
    if (def.w.u !== undefined) { const c = def.w.u*uVal; weight += c;
      if (c > topContribution) { topContribution = c; topFactor = "urgency"; } }
    if (def.w.r !== undefined) { const c = def.w.r*rVal; weight += c;
      if (c > topContribution) { topContribution = c; topFactor = "reply status"; } }

    results.push({ name, weight, topFactor });
  }

  results.sort((a,b) => b.weight - a.weight);
  return results; // results[0] is the recommended action
}

📌 What else we can do (next steps)

  • Confidence display — show the winning action's weight gap over the runner-up
  • Feedback loop — log which suggestions reps act on, to inform Phase 3's ML model
  • Snooze / dismiss — let reps override, feeding a correction signal back in
  • Time-decay — recent activity should count more than a reply from weeks ago
  • Ranked list — show top 2–3 candidate actions instead of just one
